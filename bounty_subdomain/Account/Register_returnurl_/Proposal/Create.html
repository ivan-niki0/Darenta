Darenta Marketing Bounty Program

The Darenta team actively encourages the community to be involved with Darenta marketing.

We welcome and reward good writers and artists for our bounty program.

Published articles bounties:

if you have published an article. Please contact us for a reward.

50 tokens for self published articles.

750 tokens for articles published in mainstream media.

Youtube Videos:

if you have published a youtube video. Please contact us for a reward, and we can add it to our community youtube channel.

50 tokens for a basic video.

Up to 500 tokens for the most well produced videos.

Blog posts for the Darenta Blog:

Rewards - 100 to 1000 tokens.

The best way to write a blog, which is the language used to write posts in the Darenta forum and is easy to use. Please make sure your pics/infographics have proper hosting links on a third party hosting site. A guide to write accordingly can be found here.

The best way to share with the Darenta team is to paste the code into a Google doc and share that.

Blogs can be written like articles that discuss a topic or “how to” tutorials.

Developer Bounty Program:

Darenta’s developer bounties offer rewards for all proficiencies. From the experienced Dev to the aspiring student, there is ProducToken to be earned.

Our Bounties range from detecting bugs, to building apps on top on Darenta.

Bug bounties can be earned by following the steps listed here. For large scale application or module bounties, fill out the contact sheets below.

Development Project Bounties:

Rewards - 500 to 10000 tokens.

Requirements:

a working demo written in ES6 Javascript running on nodejs (preferred language). Depending on the project scope and with prior discussion, projects can also be written in Python, C++, C#, PHP, Java, GO.

Open-sourced code on Github under an MIT license.

A detailed “how to” blog explaining exactly how the project was built and walking through others on how to build it as well as a tutorial for users will receive higher bounties.

Development Bug Bounties:

Rewards - 50 tokens and up.

Requirements:

to receive a bug bounty, simply post the detailed issue. These must be made in Github and have the payout address included. The issue must not be a duplicate of an already submitted issue.

There will be an additional amount paid for a pull request that is accepted per each bug.

If you experience a problem, try to make a detailed description (report your device, OS version and app version) and take screen shots if that helps understanding the issue. In order to receive PROD, you must report the issue on github, accompanied with your eth address.

There will be an additional amount paid for a pull request that is accepted per each bug.
